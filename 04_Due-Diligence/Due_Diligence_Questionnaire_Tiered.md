# Vendor Due Diligence Questionnaire (Tiered) — US Privacy + Cloud Data

**Instructions:** Answer all items for your tier and attach evidence references (EV-###).

## A) Vendor profile (All tiers)
1) Describe the service and data processing purpose.
2) Provide security and privacy contacts.
3) List subprocessors and where data is processed/stored.

## B) Privacy program (US-aligned) (Medium+)
4) Describe how you limit data use to purpose (purpose limitation).
5) Data minimization: what data is required vs optional?
6) Retention: define retention schedule and deletion process.
7) Rights readiness: can you support deletion/access/correction requests when required by contract/customer obligations?
8) Service provider/processor commitments: confirm no sale/sharing of personal data outside contracted purpose.
9) Employee access to personal data: access controls and least privilege.

## C) Cloud security (Medium+)
10) Encryption in transit and at rest (algorithms not required; confirm controls).
11) Key management approach (vendor-managed vs customer-managed options if available).
12) Tenant isolation and access boundaries.
13) Administrative access controls (MFA/SSO, privileged access management).
14) Logging: access logs, admin logs, security event logging and retention.

## D) Security governance (High/Critical)
15) Security policies: provide index or summary (acceptable evidence types defined in Evidence Criteria).
16) Risk management approach: how risks are identified and tracked.
17) Vulnerability management: scanning cadence, patch SLAs, critical fix timelines.
18) Secure SDLC (if software): code review, CI/CD controls, dependency controls.
19) Independent testing: pen test summary or equivalent.

## E) Incident response + breach readiness (High/Critical)
20) Incident response plan exists (Y/N) + last exercised date.
21) Breach notification commitment: timeline and contact path.
22) Ransomware and data exfiltration response approach (high-level).

## F) Business resilience (High/Critical)
23) BC/DR plan and tested recovery evidence.
24) RTO/RPO targets (or typical restoration expectations).

## G) Subprocessors/fourth parties (High/Critical)
25) Subprocessor onboarding controls and reassessment cadence.
26) Flow-down contractual requirements to subprocessors.
27) Notification process for subprocessor changes.

## H) Assurance artifacts (Preferred for High/Critical)
28) SOC 2 Type II / ISO 27001 / equivalent evidence pack (attach or provide bridge letter summary if limited).
