# TPRM Standard — Control Requirements by Tier (Summary)

This standard defines baseline control requirements. Detailed control IDs live in:
`05_Control-Library-and-Mapping/Unified_Control_Library_US_Cloud.md`

## Tier: Low
- Data handling statement
- Confidentiality / NDA terms
- Incident contact + notification commitment
- Minimal access principle documented (no privileged access)

## Tier: Medium
Low +:
- MFA/SSO for vendor access
- Encryption in transit and at rest
- Logging for access and admin actions (where applicable)
- Vulnerability management cadence
- Data retention and deletion commitment

## Tier: High
Medium +:
- Documented security program (policies + governance)
- Incident response plan + tabletop or evidence of readiness
- Subprocessor disclosure + controls
- BC/DR plan and RTO/RPO targets or statement
- Secure SDLC (if vendor provides software/services)

## Tier: Critical
High +:
- Formal assurance evidence (SOC 2 Type II / ISO cert / equivalent evidence pack)
- Independent testing evidence (pen test summary / vulnerability assessment)
- Contractual audit rights / security annex acceptance
- Strong key management expectations (where applicable)
- Time-bound remediation for findings + executive sign-off
