# US Privacy + Cloud Security Contract Checklist (Non-Legal, Practical)

## Data processing terms
- Purpose limitation (no use beyond contract)
- Data minimization and scope of data types
- Retention schedule + deletion/return at termination
- Subprocessor disclosure + change notification + flow-down obligations

## US privacy expectations (service provider style)
- No sale/share of personal data outside contracted purpose
- Support for contractual rights requests (delete/access/correct) where applicable
- Restrictions on secondary use and marketing reuse

## Security safeguards
- Encryption in transit + at rest
- MFA/SSO for access; privileged access controls
- Logging + retention
- Vulnerability management expectations + patch timelines

## Incident/breach
- Incident response plan commitment
- Notification obligation (defined timeframe) + escalation contacts
- Cooperation and forensics support clause

## Audit/assurance
- Right to request assurance evidence (SOC2/ISO) or equivalent evidence pack
- Security questionnaire/evidence cooperation clause

## Cloud specifics
- Data location/residency statement
- Tenant isolation expectations (where applicable)
- Key management responsibilities (shared responsibility clarity)

> This checklist supports contract review; final language should be validated by Legal.
