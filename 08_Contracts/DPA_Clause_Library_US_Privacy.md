# DPA / Security Annex Clause Library (US Privacy + Cloud)
**Not legal advice.** This is a portfolio-grade clause checklist to support contracting discussions.

## A) Purpose limitation
- Vendor processes data only to provide the contracted services.
- No secondary use (marketing, profiling, resale) without written authorization.

## B) Data minimization
- Vendor collects/processes only necessary fields.
- Any optional fields require customer approval.

## C) Retention + deletion/return
- Retention schedule documented.
- Deletion/return at termination with verifiable confirmation.
- Deletion SLA and method (including subprocessors).

## D) Subprocessors / fourth parties
- Subprocessor list disclosure.
- Flow-down obligations required.
- Time-bound change notification window (e.g., 30 days) + right to object for High/Critical.
- Clear responsibility for subprocessors.

## E) Security safeguards (baseline)
- Encryption in transit + at rest.
- SSO/MFA for access; privileged access controls.
- Logging + retention requirements.
- Vulnerability management + patch timelines.

## F) Incident / breach
- Defined notification timeline (time-bound).
- Cooperation, forensic preservation, and support.
- Root cause and corrective action summary on request.

## G) Audit/assurance cooperation
- Vendor provides SOC2/ISO evidence (or equivalent) for High/Critical.
- Vendor supports questionnaires/evidence requests.

## H) Data location / cloud specifics
- Data residency commitments (if required).
- Tenant isolation assurances.
- Shared responsibility clarity (who secures what).
